作为一个编程的人，时刻面临者代码管理的问题，当你的代码随着你的进度不断变多，但又不想放弃以前的版本的时候，对于其他人来说可能会选择保存多次来记录进度，但是git座椅版本控制工具就能为版本管理提供帮助。
当我们需要做版本的控制，进度的记录，团队的开发等复杂工作的时候，git能够很好的帮我们理清思路，做出相应的记录。占用极少的空间，达到最大的效率。当然，如果我们在此基础上能够有一个专门存放代码的地方，让代码不与其他文件重合，互相干扰，想必是极好的。由此诞生了托管中心的概念。托管中心可以理解成百度云这样的工具，但是只做代码管理，在内网与外网有不同的选择。

  • 内网：可以自己搭建一个GitLab服务器
    • 外网：可以使用码云、Github

而git也不是唯一的版本控制工具，以集中式与分布式区分：

  • 集中式：CSV ,SVN,VSS
  • 分布式：Git，Darcs,…

分支管理

hot_fix master feature_x feature_y

什么是分支管理

  • 在版本控制中，使用推进多个任务，在分支管理的好处当中，我们保留我们的主进度，在分支当中开发我们的程序，完成之后并入主进度，这样能够防止文件开发过程当中出现混乱影响主要进度的开发。

分支的好处

  • 同时并行推进多个功能开发，提高开发效率
  • 某一分支开发失败，不会对其它分支有任何影响

Git 结合Github

github介绍：GitHub是一个面向开源及私有软件项目的托管平台，因为只支持git 作为唯一的版本库格式进行托管，故名GitHub。
GitHub于2008年4月10日正式上线，除了Git代码仓库托管及基本的 Web管理界面以外，还提供了订阅、讨论组、文本渲染、在线文件编辑器、协作图谱（报表）、代码片段分享（Gist）等功能。目前，其注册用户已经超过350万，托管版本数量也是非常之多，其中不乏知名开源项目 Ruby on Rails、jQuery、python 等。
来源：百度百科
你可以理解成这是一个只支持存储项目文件的云端，且能够通过git远程操作。在进行以下操作之前，请先保证有一个GitHub账号，并且创建了一个仓库。

推送

克隆(下载)

完整的把远程库克隆(下载)到本地 克隆下来后不要在主分支里面做开发 clone进行一次，从无到有的过程，更新用pull

解决冲突

注意：解决冲突后的提交是不能带文件名的

如果不是基于远程库最新版做的修改不能推送，必须先pull下来安装冲突办法解决

rebase(变基)

rebase的目的是让git记录更加简洁

  • 1、将多个提交记录整合成一个记录
  • 版本号
  • 数量 注：回车之后有提示信息，根据提示信息进行修改，尽量合并那些还没有被合并到版本库当中的文件。
  • 2、将分支融入进master当中
    当开发的时候生成了一个分支，主分支也在同步开发，然后两个分支合并在一起，在用图形查看的时候，就会发现图上多了一个分支，然后融合会主线的过程，开发显得不再是一条线了，当分支越多，这个过程越为复杂。因此为了使得记录显示高效，需要想办法把分支完全融入主线。
  • 3、当远程仓库进度超越本地进度的时候
    当远程仓库超越本地记录，直接pull会令开发产生分叉，因此不能用pull
